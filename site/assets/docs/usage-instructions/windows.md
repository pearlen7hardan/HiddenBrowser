# Windows

## Programs for proxy setup on Windows

Clients for routing traffic through a SOCKS5 / HTTP proxy. Pick one, import your proxy, and point the app at it.

### [<mark style="color:purple;">**ClashX**</mark>](https://github.com/lantongxue/clash_for_windows_pkg)

Desktop GUI for the rule-based Clash proxy core — routes traffic through SOCKS5 / HTTP with custom rules and UDP. Open-source (the project is archived, but existing builds still work).

* UDP support: Yes
* Difficulty: High
* Split routing: Yes

### [<mark style="color:purple;">**V2RayN**</mark>](https://github.com/2dust/v2rayN)

Widely used open-source Windows GUI for the Xray / V2Ray cores. Handles SOCKS5 / HTTP with UDP and per-route rules.

* UDP support: Yes
* Difficulty: Medium
* Split routing: Yes

### [<mark style="color:purple;">**NekoRay**</mark>](https://github.com/MatsuriDayo/nekoray)

Open-source Qt proxy GUI (nekoray / NekoBox for PC) built on the sing-box and Xray cores. Supports UDP and routing.

* UDP support: Yes
* Difficulty: High
* Split routing: Yes

### [<mark style="color:purple;">**ProxyCap**</mark>](https://www.proxycap.com/)

Commercial utility that forces any application's traffic through a SOCKS5 / HTTP proxy by rules. Supports UDP.

* UDP support: Yes
* Difficulty: Medium
* Split routing: Yes

### [<mark style="color:purple;">**SocksCap64**</mark>](https://github.com/bobo2334/sockscap64)

Free tool that tunnels selected Windows applications through a SOCKS5 proxy (per-app). No UDP.

* UDP support: No
* Difficulty: Low
* Split routing: Yes

### [<mark style="color:purple;">**Proxifier**</mark>](https://www.proxifier.com/)

Commercial proxifier that routes any app through a proxy with flexible rules. Paid; no UDP.

* UDP support: No
* Difficulty: Low
* Split routing: Yes
* Paid
